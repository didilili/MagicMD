# v0.5 Roadmap

v0.4.0 已完成发布，重点是 CLI 默认中文优先、官网中英文文档、配置生成器语言选项、Agent Skill 使用说明，以及格式化和 commit 规范。

v0.5 建议继续沿着“中文文章归档到 Markdown 内容包”这个产品方向，把真实中文站点回归、Agent 使用入口和发布流程稳定性补强。

## Goals

- 建立更系统的真实站点回归集，优先覆盖微信公众号、掘金和 CSDN 的中文技术文章。
- 让 Agent 使用体验更像产品入口：用户可以用一句话完成安装、转换、批量整理和结果检查。
- 优化官网首页和快速开始，让新用户更快理解 MagicMD 的核心场景和安装路径。
- 增加发布后 smoke test checklist，覆盖 PyPI、npm、GitHub Release 和 Skill 安装路径。
- 评估 PyPI Trusted Publisher 和 npm GitHub Actions 发布流程，减少手动 token 发布和权限切换风险。

## Proposed Tasks

- 设计真实站点回归清单结构，明确 URL、平台、预期检查点、人工复核字段和隐私处理规则。
- 为微信公众号、掘金、CSDN 各补充一组代表性中文技术文章样本。
- 增加回归运行说明：如何批量转换、检查 `batch-report.md`、抽查 `article.md` 和保留失败证据。
- 强化 Skill 文档中的自然语言使用示例，覆盖安装、单篇转换、批量转换、结果检查和失败排查。
- 优化官网首页文案，把“中文文章归档到 Markdown 内容包”作为第一屏核心价值。
- 新增发布后 checklist 文档，记录 PyPI、npm、GitHub Release、Skill 安装路径和 smoke test 命令。
- 调研并记录 PyPI Trusted Publisher 与 npm GitHub Actions 发布方案，先形成维护者决策文档。

## Non-goals

- v0.5 不优先新增大平台适配，除非真实回归暴露出必须修复的平台解析问题。
- v0.5 不把 MagicMD 改造成通用爬虫框架。
- v0.5 不把发布到 GitHub 仓库作为核心流程实现；发布工作流继续放到 v0.6 评估。

## Success Criteria

- 发布后 smoke test 能稳定确认 `magicmd==0.5.0` 和 `magicmd@0.5.0` 可安装。
- 官网首页和快速开始能让中文用户在 3 分钟内理解安装、转换和输出结构。
- Agent Skill 文档包含可直接复制给 Codex、Claude Code 等工具的一句话示例。
- 真实站点回归集能帮助维护者发现格式退化，而不是只依赖单元测试。

# v0.5 Roadmap

v0.4.0 shipped Chinese-first CLI output, bilingual website docs, a Config Builder language option, Agent Skill usage guidance, and formatting / commit conventions.

v0.5 should keep pushing MagicMD toward its core product direction: archiving Chinese articles into durable Markdown content packages, with stronger live regression coverage, agent entrypoints, and release reliability.

## Goals

- Build a more systematic live-site regression corpus, prioritizing Chinese technical articles from WeChat, Juejin, and CSDN.
- Make the Agent experience feel like a product entrypoint: users should be able to install, convert, batch organize, and inspect results with short natural-language requests.
- Improve the website homepage and quick start so new users quickly understand MagicMD's core scenario and install paths.
- Add a post-release smoke test checklist for PyPI, npm, GitHub Release, and Skill installation paths.
- Evaluate PyPI Trusted Publisher and npm GitHub Actions publishing to reduce manual token and permission friction.

## Proposed Tasks

- Design the live regression manifest structure, including URL, platform, expected checks, manual review fields, and privacy rules.
- Add representative Chinese technical article samples for WeChat, Juejin, and CSDN.
- Document how to run regression batches, inspect `batch-report.md`, spot-check `article.md`, and preserve failure evidence.
- Strengthen natural-language Skill examples for installation, single conversion, batch conversion, result inspection, and failure troubleshooting.
- Refine homepage copy around the first-screen promise: archiving Chinese articles into Markdown content packages.
- Add a post-release checklist covering PyPI, npm, GitHub Release, Skill install paths, and smoke test commands.
- Research PyPI Trusted Publisher and npm GitHub Actions publishing, then capture the maintainer decision before implementation.

## Non-goals

- v0.5 does not prioritize large new platform adapters unless live regression reveals parser issues that must be fixed.
- v0.5 does not turn MagicMD into a general-purpose crawler framework.
- v0.5 does not implement GitHub publishing as a core flow; publishing workflow exploration remains a v0.6 topic.

## Success Criteria

- Post-release smoke tests can confirm `magicmd==0.5.0` and `magicmd@0.5.0` install successfully.
- The website homepage and quick start help Chinese users understand installation, conversion, and output structure within 3 minutes.
- Agent Skill docs include copyable one-sentence examples for Codex, Claude Code, and similar tools.
- The live-site regression corpus helps maintainers catch formatting regressions instead of relying only on unit tests.
