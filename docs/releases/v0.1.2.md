# MagicMD v0.1.2

v0.1.2 是一个很小但很关键的发布版本：MagicMD 的 Agent Skill 从“仓库里有说明”变成“有正式可安装目录”。

如果你用 Codex、Claude Code 或其他支持 Skill 的 Agent 工具，这一版让 Agent 更容易找到 MagicMD 的入口，也让后续发布不容易误删 Skill 文件。

## 这版处理的事情

- 新增 `skills/magicmd/SKILL.md`，作为正式可安装的 MagicMD Skill。
- 新增 `skills/magicmd/agents/openai.yaml`，补充 Skill 在 Agent UI 里的显示信息。
- 保留根目录 `SKILL.md` 作为兼容入口，并指向正式 Skill 目录。
- README 和开发文档改为使用 `skills/magicmd` 作为安装路径。
- 新增测试，确保 Skill 发布目录不会被误删。

## 安装 Skill

从 GitHub 安装时使用：

```text
Repository: didilili/MagicMD
Skill path: skills/magicmd
```

如果使用 Codex 的 skill installer，可以运行：

```bash
python3 ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --repo didilili/MagicMD \
  --path skills/magicmd
```

安装后重启 Codex，让新 Skill 生效。

## 安装这个版本

```bash
uv tool install magicmd==0.1.2
# 或
pipx install "magicmd==0.1.2"
# npm wrapper
npm install -g magicmd@0.1.2
```

# MagicMD v0.1.2

v0.1.2 makes the MagicMD Agent Skill available from a clean installable directory.

The article conversion behavior does not change in this release. The main change is packaging: agents can now install the Skill from `skills/magicmd`, while the root `SKILL.md` remains as a compatibility entrypoint.

## What Changed

- Added `skills/magicmd/SKILL.md` as the publishable Skill entrypoint.
- Added `skills/magicmd/agents/openai.yaml` for Agent UI metadata.
- Kept the root `SKILL.md` as a compatibility pointer.
- Updated docs to use `skills/magicmd` as the install path.
- Added a metadata test to protect the Skill directory.

## Install This Version

```bash
uv tool install magicmd==0.1.2
# or
pipx install "magicmd==0.1.2"
# npm wrapper
npm install -g magicmd@0.1.2
```
