# MagicMD v0.7.0

v0.7.0 解决的是：不熟悉命令行的用户，怎样更直观地把公开文章转换成 Markdown 内容包，并在真正发布前看清楚 GitHub 发布计划。

这一版新增 MagicMD Studio 本地网页控制台。它不是另一套转换引擎，而是把已有 CLI、SDK、配置生成和发布预览能力放进一个本地界面里：用户可以粘贴单篇文章链接、批量转换、保存配置、打开输出目录，并预览发布到 GitHub 内容仓库的计划。

这条线也延续了之前几个版本的节奏：

- v0.1：稳定转换公开文章。
- v0.2：让输出适配不同发布系统。
- v0.3：提供 Python SDK，方便业务系统直接调用。
- v0.4：补齐中文优先、Agent 友好的上手体验。
- v0.5：让归档结果更完整，发布维护更稳。
- v0.6：把转换结果发布到 GitHub 内容仓库。
- v0.7：给转换与发布预览补上本地可视化控制台。

## 这版新增了什么

- 新增 `magicmd studio` 命令，启动本地 MagicMD Studio 控制台。
- Studio 支持单篇转换、批量转换、输出目录打开和转换结果提示。
- Studio 支持在界面里生成并保存 `.magicmd.toml` 配置。
- Studio 支持读取 `[publish.github]` 配置并预览 GitHub 发布计划。
- 网站新增 Studio 文档入口，中英文快速开始都改为先引导用户打开 Studio。
- Python wheel 打包 Studio 静态资源，安装后即可直接运行本地控制台。

## 稳定性修复

- 修复部分 Markdown 加粗标签后紧贴正文时，GFM/CommonMark 不能稳定解析的问题。
- 现在类似 `**① headroom:**压缩 LLM 输入` 会规范成 `**① headroom:** 压缩 LLM 输入`。
- 新增 `broken_strong_boundary` 质量扫描，批量转换报告可以识别这类坏边界。
- 保留 `**clientX**、**clientY**`、`**AI**时代` 等正常相邻加粗写法。

## 推荐用法：先打开 Studio

安装或升级后，直接运行：

```bash
magicmd studio
```

浏览器会打开本地控制台：

```text
http://127.0.0.1:8765/
```

如果你只想转换文章，粘贴公开文章链接后点击“转换 Markdown”。如果你已经配置了 GitHub 发布目标，可以先点击“预览发布计划”，确认目标路径、文件列表和分支策略后，再决定是否用 CLI 真实发布。

## 安全边界

- Studio 默认监听 `127.0.0.1`，面向本机使用。
- 本地接口带启动期 token，前端请求必须携带 token。
- 打开路径时限制在 Studio 工作目录内，避免误打开任意系统路径。
- GitHub 发布预览不会推送远端；真实发布仍由 `magicmd publish github` 执行。

# MagicMD v0.7.0

v0.7.0 adds MagicMD Studio, a local web console for converting public articles into Markdown packages and previewing GitHub publishing plans before any remote write.

Studio is not a second conversion engine. It wraps the existing CLI, SDK, configuration builder, and publishing preview workflow in a local interface, so users can convert one article, run batch conversion, save `.magicmd.toml`, open output folders, and inspect publishing plans without memorizing every command.

## What Changed

- Added the `magicmd studio` command.
- Added local MagicMD Studio for single conversion, batch conversion, config generation, output-folder opening, and GitHub publishing previews.
- Added bilingual Studio documentation and quick-start links.
- Packaged Studio static assets in the Python wheel.
- Added startup token checks and workspace path limits for local Studio requests.

## Fixes

- Normalized Markdown strong-label boundaries so GFM/CommonMark can parse bold labels reliably.
- Added `broken_strong_boundary` quality scanning for future batch conversion reports.
- Kept valid adjacent bold text such as `**clientX**、**clientY**` and `**AI**时代` unchanged.

## Recommended Flow

After installing or upgrading, start the local console:

```bash
magicmd studio
```

Studio opens at:

```text
http://127.0.0.1:8765/
```

Use it to convert public article URLs, review generated package files, save configuration, and preview GitHub publishing plans. Real GitHub publishing still runs through `magicmd publish github`.
