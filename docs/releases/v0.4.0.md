# MagicMD v0.4.0

v0.4.0 是 MagicMD 继续走向“中文优先、Agent 友好、上手更顺”的版本。

这一版不改变核心转换目标：MagicMD 仍然专注把公开文章链接转换成可长期保存的 Markdown 内容包。重点变化在体验层：CLI 默认中文提示，官网补齐中英文文档，配置生成器可以设置终端语言，Agent Skill 的安装和使用说明也更清楚。

## 这版新增了什么

- CLI 终端提示默认改为中文优先，更符合 MagicMD 当前的主要使用场景。
- 新增 `[ui] language = "en-US"` 配置，英文用户仍可以显式切回英文进度和结果提示。
- 配置生成器新增终端语言选项；这个选项只影响 CLI 输出语言，不会改动 Markdown front matter 字段名或来源信息块模板。
- 官网新增中英文双语文档入口，补齐快速开始、配置、SDK、支持站点、故障排查和 Agent Skill 使用说明。
- Agent Skill 文档不再只围绕 Codex 描述，也适用于 Claude Code 等支持 Skill 的 Agent 工具。
- Skill 安装说明新增“一句话描述安装需求”的方式，适合直接交给 Agent 执行，同时保留手动安装命令作为兜底。
- 新增 Ruff、Prettier、lint-staged、Husky 和 commitlint 工作流，提交时自动格式化 staged 文件，并继续规范 commit message。

## 使用示例

中文用户可以直接运行：

```bash
magicmd "https://mp.weixin.qq.com/s/example"
```

如果需要英文终端输出，可以在 `.magicmd.toml` 中配置：

```toml
[ui]
language = "en-US"
```

如果希望 Agent 安装 MagicMD Skill，可以直接描述：

```text
请从 GitHub 仓库 didilili/MagicMD 安装 MagicMD Skill，Skill 路径是 skills/magicmd。
```

# MagicMD v0.4.0

v0.4.0 makes MagicMD more Chinese-first, agent-friendly, and easier to adopt.

The conversion goal stays the same. This release improves the surrounding experience: Chinese-first CLI messages, bilingual website docs, terminal-language configuration, clearer Agent Skill installation guidance, and formatting/linting workflows for maintainers.

## What Changed

- Made CLI messages Chinese-first by default.
- Added `[ui] language = "en-US"` for English CLI output.
- Added terminal-language configuration to the Config Builder.
- Added bilingual website docs for quick start, config, SDK usage, supported sites, troubleshooting, and Agent Skill usage.
- Generalized Agent Skill docs beyond Codex.
- Added a simple agent-facing Skill install request.
- Added Ruff, Prettier, lint-staged, Husky, and commitlint workflows.
