# MagicMD v0.4.0

v0.4.0 是 MagicMD 继续走向“中文优先、Agent 友好、上手更顺”的版本。

这次升级不改变核心转换目标：MagicMD 仍然专注把公开文章链接转换成可长期保存的 Markdown 内容包。重点变化在体验层：CLI 默认中文提示，官网补齐中英文文档，配置生成器能设置终端语言，Agent Skill 的安装和使用说明也更清楚。

## 这次升级了什么

- CLI 终端提示默认改为中文优先，更符合 MagicMD 当前的主要使用场景。
- 新增 `[ui] language = "en-US"` 配置，英文用户仍可以显式切回英文进度和结果提示。
- 配置生成器新增终端语言选项；这个选项只影响 CLI 输出语言，不会改动 Markdown front matter 字段名或来源信息块模板。
- 官网新增中英文双语文档入口，补齐快速开始、配置、SDK、支持站点、故障排查和 Agent Skill 使用说明。
- Agent Skill 文档不再只围绕 Codex 描述，也适用于 Claude Code 等支持 Skill 的 Agent 工具。
- Skill 安装说明新增“一句话描述安装需求”的方式，适合直接交给 Agent 执行，同时保留手动安装命令作为兜底。
- 新增 Ruff、Prettier、lint-staged、Husky 和 commitlint 工作流，提交时自动格式化 staged 文件，并继续规范 commit message。

## 为什么值得升级

v0.1 解决的是“能稳定转文章”。

v0.2 解决的是“输出能适配你的发布系统”。

v0.3 解决的是“业务系统能直接调用 MagicMD”。

v0.4 解决的是“中文用户和 Agent 用户更容易开始使用 MagicMD”。

如果你主要在中文内容平台做归档，升级后默认终端提示会更自然：

```bash
magicmd "https://mp.weixin.qq.com/s/example"
```

如果你需要英文终端输出，可以在 `.magicmd.toml` 中配置：

```toml
[ui]
language = "en-US"
```

如果你希望 Agent 安装 MagicMD Skill，可以直接描述：

```text
请从 GitHub 仓库 didilili/MagicMD 安装 MagicMD Skill，Skill 路径是 skills/magicmd。
```

# MagicMD v0.4.0

v0.4.0 makes MagicMD more Chinese-first, agent-friendly, and easier to adopt.

This release keeps the core conversion goal unchanged: MagicMD still focuses on turning public article URLs into durable Markdown content packages. The main improvements are in the user experience: Chinese-first CLI messages, bilingual website docs, a terminal-language option in the Config Builder, and clearer Agent Skill installation guidance.

## What's Changed

- CLI terminal messages now default to Chinese-first output, matching MagicMD's primary usage scenario.
- Added `[ui] language = "en-US"` so English users can keep English progress and result messages.
- Added a terminal-language option to the Config Builder. This only affects CLI output and does not rename Markdown front matter fields or the source block template.
- Added bilingual website documentation for quick start, config, SDK usage, supported sites, troubleshooting, and Agent Skill usage.
- Generalized the Agent Skill guide beyond Codex so it also fits Claude Code and other Skill-capable agent tools.
- Added a one-sentence Skill installation request that users can give directly to an agent, while keeping the manual command as a fallback.
- Added Ruff, Prettier, lint-staged, Husky, and commitlint workflows so staged files are formatted before commit while commit messages stay conventional.

## Why Upgrade

v0.1 made article conversion reliable.

v0.2 made output configurable for publishing systems.

v0.3 made MagicMD callable from backend applications.

v0.4 makes MagicMD easier to start with for Chinese users and agent-driven workflows.

For Chinese-first CLI output:

```bash
magicmd "https://mp.weixin.qq.com/s/example"
```

For English CLI output, configure `.magicmd.toml`:

```toml
[ui]
language = "en-US"
```

To install the MagicMD Skill through an agent, you can say:

```text
Install the MagicMD Skill from GitHub repo didilili/MagicMD, using Skill path skills/magicmd.
```
