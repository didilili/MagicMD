# MagicMD v0.3.0

v0.3.0 是 MagicMD 从“命令行工具”走向“可被业务系统稳定调用”的版本。

这一版的重点不是新增站点，而是把转换能力沉淀成公开 Python SDK。CLI、npm wrapper 和 Agent Skill 继续保留；外部 Python 项目现在可以直接 import MagicMD，把公开文章转换成结构化结果，再写入自己的数据库、媒体目录或发布流程。

## 这版新增了什么

- 新增公开 API：`from magicmd import convert_article`。
- 新增 `ArticleConversionResult`，返回标题、作者、平台、来源链接、发布时间、摘要、Markdown、内容 hash、图片资产、warning、metadata、转换报告和可选内容包目录。
- 新增 `ConvertedImage`，用于描述图片资产。
- 明确图片路径语义：`markdown_path` 是 Markdown 中实际引用的路径；`local_path` 是本地已下载图片的文件系统路径。
- 新增 SDK 错误类型：`UnsupportedPlatformError`、`FetchError`、`ParseError`、`MediaDownloadError` 和 `ConversionError`。
- 支持 `output_dir=None` 的纯内存转换，不强制写出内容包。
- 传入 `output_dir` 时仍保留 CLI 内容包写出行为。
- CLI 单篇转换流程改为复用 SDK，避免 CLI 和 Python API 维护两套转换逻辑。
- README / README_EN 新增 Python SDK 使用示例。

## Python SDK 示例

纯内存转换：

```python
from magicmd import convert_article

result = convert_article(
    url="https://mp.weixin.qq.com/s/example",
    platform="auto",
    output_dir=None,
)

print(result.title)
print(result.markdown)
print(result.images)
```

同时写出内容包：

```python
result = convert_article(
    url="https://juejin.cn/post/example",
    output_dir="output",
)

print(result.package_dir)
print(result.report)
```

按阶段捕获错误：

```python
from magicmd import FetchError, ParseError, UnsupportedPlatformError, convert_article

try:
    result = convert_article("https://mp.weixin.qq.com/s/example")
except UnsupportedPlatformError:
    ...
except FetchError:
    ...
except ParseError:
    ...
```

# MagicMD v0.3.0

v0.3.0 turns MagicMD from a CLI-first tool into a library that Python applications can call directly.

The main addition is the public `convert_article()` API. Python projects can now convert public article URLs, receive structured metadata and Markdown, and decide whether to keep everything in memory or write a full content package.

## What Changed

- Added `from magicmd import convert_article`.
- Added `ArticleConversionResult` for structured conversion output.
- Added `ConvertedImage` for image metadata.
- Clarified downloaded image paths versus Markdown reference paths.
- Added explicit SDK errors for platform detection, fetch, parse, media, and conversion failures.
- Added in-memory conversion with `output_dir=None`.
- Reused the SDK from the CLI single-conversion path.
- Added SDK examples to README / README_EN.
