# MagicMD v0.3.0

v0.3.0 是 MagicMD 从“命令行工具”走向“可被业务系统稳定调用”的版本。

这次升级的重点不是新增站点，而是把转换能力沉淀成公开 Python SDK。CLI、npm wrapper 和 Agent Skill 仍然保留，但外部 Python 项目现在可以直接 import MagicMD，把公开文章转换成结构化结果，再写入自己的数据库、媒体目录或发布流程。

## 这次升级了什么

- 新增公开 API：`from magicmd import convert_article`。
- 新增 `ArticleConversionResult`，返回标题、作者、平台、来源链接、发布时间、摘要、Markdown、内容 hash、图片资产、warning、metadata、转换报告和可选内容包目录。
- 新增 `ConvertedImage`，用于描述图片资产。
- 明确图片路径语义：`markdown_path` 是 Markdown 中实际引用的路径；`local_path` 是本地已下载图片的文件系统路径。
- 新增清晰的 SDK 错误类型：`UnsupportedPlatformError`、`FetchError`、`ParseError`、`MediaDownloadError` 和 `ConversionError`。
- 支持 `output_dir=None` 的纯内存转换，不强制写出内容包。
- 传入 `output_dir` 时仍保留原 CLI 行为，继续生成 `article.md`、`metadata.json`、`extraction-report.json` 和媒体目录。
- CLI 单篇转换流程改为复用 SDK，避免 CLI 和 Python API 维护两套转换逻辑。
- README / README_EN 新增 Python SDK 使用示例。

## 为什么值得升级

v0.1 解决的是“能稳定转文章”。

v0.2 解决的是“输出能适配你的发布系统”。

v0.3 解决的是“业务系统能直接调用 MagicMD”。

比如 Web 后端可以直接这样用：

```python
from magicmd import convert_article

result = convert_article(
    url="https://mp.weixin.qq.com/s/example",
    platform="auto",
    output_dir=None,
)

print(result.title)
print(result.markdown)
print(result.images)
```

如果需要同时生成内容包：

```python
result = convert_article(
    url="https://juejin.cn/post/example",
    output_dir="output",
)

print(result.package_dir)
print(result.report)
```

错误也可以按阶段捕获：

```python
from magicmd import FetchError, ParseError, UnsupportedPlatformError, convert_article

try:
    result = convert_article("https://mp.weixin.qq.com/s/example")
except UnsupportedPlatformError:
    ...
except FetchError:
    ...
except ParseError:
    ...
```

# MagicMD v0.3.0

v0.3.0 moves MagicMD from a CLI-first tool toward a library that Python projects can call directly.

This release does not focus on adding more sites. It turns the existing conversion pipeline into a public Python SDK. The CLI, npm wrapper, and Agent Skill remain available, while Python applications can now import MagicMD, convert public article URLs, and store the structured result in their own database, media pipeline, or publishing workflow.

## What's Changed

- Added the public API: `from magicmd import convert_article`.
- Added `ArticleConversionResult` with title, author, platform, source URL, publish time, excerpt, Markdown, content hash, image assets, warnings, metadata, extraction report, and optional package directory.
- Added `ConvertedImage` for image asset metadata.
- Clarified image path semantics: `markdown_path` is the path referenced by Markdown; `local_path` is the downloaded filesystem path.
- Added explicit SDK errors: `UnsupportedPlatformError`, `FetchError`, `ParseError`, `MediaDownloadError`, and `ConversionError`.
- Added in-memory conversion with `output_dir=None`.
- Kept package writing when `output_dir` is provided, including `article.md`, `metadata.json`, `extraction-report.json`, and media directories.
- Updated the CLI single-conversion path to reuse the SDK, avoiding duplicate conversion logic.
- Added Python SDK examples to README / README_EN.

## Why Upgrade

v0.1 made article conversion reliable.

v0.2 made output configurable for publishing systems.

v0.3 makes MagicMD callable from backend applications.

For example:

```python
from magicmd import convert_article

result = convert_article(
    url="https://mp.weixin.qq.com/s/example",
    platform="auto",
    output_dir=None,
)

print(result.title)
print(result.markdown)
print(result.images)
```

To also write a package:

```python
result = convert_article(
    url="https://juejin.cn/post/example",
    output_dir="output",
)

print(result.package_dir)
print(result.report)
```

Backend code can catch failures by stage:

```python
from magicmd import FetchError, ParseError, UnsupportedPlatformError, convert_article

try:
    result = convert_article("https://mp.weixin.qq.com/s/example")
except UnsupportedPlatformError:
    ...
except FetchError:
    ...
except ParseError:
    ...
```
