# MagicMD v0.5.0

v0.5.0 是 MagicMD 从“更容易开始使用”继续走向“更适合真实归档和发布维护”的版本。

这一版重点放在三个方向：输出更完整、回归更可控、发布流程更稳定。MagicMD 仍然专注公开文章链接到 Markdown 内容包的转换，但现在可以额外生成 Word 文档，保留微信公众号封面图，并用更系统的真实样本和发布清单支撑后续迭代。

## 这版新增了什么

- 新增可选 DOCX 导出：`magicmd convert URL --format docx` 或 `[docx] enabled = true` 会在保留 Markdown 内容包的同时生成 `article.docx`。
- DOCX 导出复用最终 `article.md`，由 Pandoc 执行，并支持配置 `pandoc_path` 和 `reference_doc`。
- 修复 `--overwrite` 覆盖同名内容包时旧图片、旧 DOCX 等生成文件可能残留的问题；现在会先清理旧包再写入本次结果。
- 新增微信公众号封面图元数据提取，并默认把封面图写入 Markdown，让归档结果更接近原文章卡片。
- 新增真实站点回归清单和 dry-run 记录，覆盖微信公众号、掘金、CSDN 等样本，方便发布前发现解析质量退化。
- 新增手动触发的 PyPI / npm Trusted Publishing workflow；默认可先做 verify dry run，再决定是否发布到 PyPI 和 npm。
- 补齐发布自动化说明和发布后 smoke test 清单，覆盖 GitHub Release、PyPI、npm 和 Agent Skill 安装路径。
- 优化官网首页、Agent Skill 文档和配置生成器体验；配置生成器在宽屏下保留更接近文档页的左右留白。

## DOCX 输出

直接生成 Markdown 内容包和 Word 文档：

```bash
magicmd convert "https://mp.weixin.qq.com/s/example" --format docx
```

如果希望长期固定输出规则，可以在 `.magicmd.toml` 中启用：

```toml
[docx]
enabled = true
pandoc_path = "pandoc"
reference_doc = ""
```

DOCX 导出复用最终的 `article.md`，不会让 Markdown 转换和 Word 转换分成两条独立逻辑。

## 发布维护

v0.5.0 开始补齐发布自动化说明和发布后 smoke test 清单。维护者可以先跑本地检查，再用 GitHub Actions 的 `Publish Release` workflow 做远端 verify：

```text
version: 0.5.0
publish_pypi: false
publish_npm: false
```

确认无误后，再决定是否发布到 PyPI 和 npm。

# MagicMD v0.5.0

v0.5.0 makes MagicMD more useful for real archiving and release maintenance.

The release focuses on richer output, more controlled regression checks, and a steadier release process. MagicMD still converts public article URLs into Markdown content packages, but it can now generate Word documents, preserve WeChat cover images, and rely on clearer live-sample and release-checklist workflows.

## What Changed

- Added optional DOCX export through `magicmd convert URL --format docx` or `[docx] enabled = true`.
- Reused the final `article.md` as the source for DOCX generation through Pandoc.
- Fixed stale generated files when `--overwrite` replaces an existing package.
- Added WeChat cover-image metadata extraction and default Markdown output.
- Added live-site regression manifests and dry-run records for WeChat, Juejin, CSDN, and related samples.
- Added manually triggered PyPI / npm Trusted Publishing workflow support.
- Added release automation notes and a post-release smoke test checklist.
- Improved the website homepage, Agent Skill docs, and Config Builder layout.
