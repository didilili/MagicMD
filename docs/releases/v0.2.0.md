# MagicMD v0.2.0

v0.2.0 是 MagicMD 从“能稳定转换”走向“能按你的发布系统输出”的版本。

这一版重点不是多加几个命令参数，而是把输出命名、Front matter、来源信息、媒体路径和 Markdown 风格整理成可配置的模板系统。你可以继续用默认配置一键转换，也可以把 MagicMD 接到自己的博客、知识库、静态站或自动化脚本里。

## 这版新增了什么

- 新增 `output.naming`，内容包目录、Markdown 文件名、metadata 文件名、提取报告文件名都可以用模板控制。
- 新增模板变量：`{title}`、`{slug}`、`{date}`、`{platform}`、`{short_hash}` 等字段可以直接用于文件名和 Markdown 输出。
- Front matter 支持自定义字段，方便接入 Hugo、Docusaurus、Hexo、个人博客和静态站。
- 正文标题和来源信息块支持模板配置，不再只能使用 MagicMD 默认格式。
- 图片和视频的 Markdown 路径支持模板，文件仍下载到本地，但正文里可以写成站点需要的路径。
- 新增 `plain`、`hugo`、`docusaurus` 三个 Markdown 预设。
- `.magicmd.example.toml` 和内置配置模板整理成可复制使用的 v0.2 示例。
- 配置错误会给出更清楚的提示，例如未知模板变量会直接说明是哪一个字段写错。

## 配置示例

把输出目录改成按日期和 slug 命名：

```toml
[output.naming]
package = "{date}-{slug}"
markdown = "index.md"
metadata = "metadata.json"
report = "extraction-report.json"
```

启用 Hugo 风格：

```toml
[markdown]
preset = "hugo"
```

自定义媒体引用路径：

```toml
[markdown.media_paths]
image = "/static/articles/{slug}/images/{filename}"
video = "/static/articles/{slug}/videos/{filename}"
```

## 怎么使用

初始化配置：

```bash
magicmd config init
```

转换单篇文章：

```bash
magicmd "https://mp.weixin.qq.com/s/xxx"
```

批量转换：

```bash
magicmd batch urls.txt -o output/
```

检查环境：

```bash
magicmd doctor
```

# MagicMD v0.2.0

v0.2.0 moves MagicMD from reliable conversion toward configurable publishing output.

This release introduces templated output naming, configurable front matter, source blocks, media paths, and Markdown presets. The goal is to make converted packages easier to drop into blogs, knowledge bases, static sites, and automation pipelines.

## What Changed

- Added `output.naming` templates for package directories and generated filenames.
- Added template variables such as `{title}`, `{slug}`, `{date}`, `{platform}`, and `{short_hash}`.
- Added configurable front matter fields.
- Added configurable document titles and source blocks.
- Added configurable media paths for images and videos.
- Added `plain`, `hugo`, and `docusaurus` Markdown presets.
- Updated the example config and packaged template.
- Improved errors for invalid template variables.

## Package Status

v0.2.0 has a GitHub tag and changelog, but no matching PyPI or npm package was published. To use these capabilities from an installable package, install v0.3.0 or later.

- GitHub tag: https://github.com/didilili/MagicMD/releases/tag/v0.2.0
- Full Changelog: https://github.com/didilili/MagicMD/compare/v0.1.2...v0.2.0
