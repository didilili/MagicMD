# MagicMD v0.2.0

v0.2.0 是 MagicMD 从“能稳定转换”走向“能按你的发布系统输出”的版本。

这一版重点不是多加几个参数，而是把输出命名、Front matter、来源信息、媒体路径和 Markdown 风格做成真正可配置的模板系统。你可以继续用默认配置一键转换，也可以把 MagicMD 接到自己的博客、知识库、静态站或者自动化脚本里。

## 这次升级了什么

- 新增 `output.naming`，内容包目录、Markdown 文件名、metadata 文件名、提取报告文件名都可以用模板控制。
- 新增模板变量：`{title}`、`{slug}`、`{date}`、`{platform}`、`{short_hash}` 等字段可以直接用于文件名和 Markdown 输出。
- Front matter 支持自定义字段，方便接入 Hugo、Docusaurus、Hexo、个人博客和静态站。
- 正文标题和来源信息块支持模板配置，不再只能使用 MagicMD 默认格式。
- 图片和视频的 Markdown 路径支持模板，文件仍下载到本地，但正文里可以写成站点需要的路径。
- 新增 `plain`、`hugo`、`docusaurus` 三个 Markdown 预设，常见发布场景可以一行启用。
- `.magicmd.example.toml` 和内置配置模板已整理成可复制使用的 v0.2 示例。
- 配置错误会给出更清晰的提示，比如未知模板变量会直接说明是哪一个字段写错。

## 为什么值得升级

v0.1 解决的是“把文章尽量转换对”。

v0.2 解决的是“转换出来以后，能不能直接进入你的发布流程”。

比如你可以把输出目录改成：

```toml
[output.naming]
package = "{date}-{slug}"
markdown = "index.md"
metadata = "metadata.json"
report = "extraction-report.json"
```

也可以启用 Hugo 风格：

```toml
[markdown]
preset = "hugo"
```

或者自定义图片引用路径：

```toml
[markdown.media_paths]
image = "/static/articles/{slug}/images/{filename}"
video = "/static/articles/{slug}/videos/{filename}"
```

## 使用方式

初始化配置：

```bash
magicmd config init
```

转换单篇文章：

```bash
magicmd "https://mp.weixin.qq.com/s/xxx"
```

批量转换：

```bash
magicmd batch urls.txt -o output/
```

检查环境：

```bash
magicmd doctor
```

# MagicMD v0.2.0

v0.2.0 moves MagicMD from "reliable conversion" toward "ready for your publishing workflow".

This release adds a real template system for output naming, front matter, source blocks, media paths, and Markdown presets. You can keep the default one-command workflow, or wire MagicMD into a blog, knowledge base, static site, or automation pipeline.

## What's Changed

- Added `output.naming` so package directories, Markdown filenames, metadata filenames, and extraction report filenames can be configured with templates.
- Added template variables such as `{title}`, `{slug}`, `{date}`, `{platform}`, and `{short_hash}`.
- Added configurable front matter fields for Hugo, Docusaurus, Hexo, personal blogs, and static sites.
- Made the document title and source block configurable instead of fixed to MagicMD defaults.
- Made image and video Markdown paths configurable while keeping downloaded media local.
- Added `plain`, `hugo`, and `docusaurus` Markdown presets.
- Updated `.magicmd.example.toml` and the packaged template so `magicmd config init` creates a ready-to-edit v0.2 config.
- Improved template errors, especially for unknown variables.

## Why Upgrade

v0.1 focused on converting articles correctly.

v0.2 focuses on making the converted output fit directly into your publishing workflow.

For example, you can configure output names:

```toml
[output.naming]
package = "{date}-{slug}"
markdown = "index.md"
metadata = "metadata.json"
report = "extraction-report.json"
```

Enable a Hugo-oriented preset:

```toml
[markdown]
preset = "hugo"
```

Or customize media paths:

```toml
[markdown.media_paths]
image = "/static/articles/{slug}/images/{filename}"
video = "/static/articles/{slug}/videos/{filename}"
```

## Usage

Initialize a config file:

Convert a batch:

