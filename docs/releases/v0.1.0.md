# MagicMD v0.1.0

这是 MagicMD 的第一个可用版本。

如果你经常把微信公众号、掘金、CSDN 里的文章整理到自己的博客、知识库或 GitHub 仓库里，这个版本已经可以开始试用了。它不会试图做成一个通用爬虫框架，而是先把“公开文章链接转 Markdown 内容包”这件事做稳。

```bash
magicmd "https://mp.weixin.qq.com/s/example"
magicmd batch urls.txt -o output/
```

一次转换会生成一个完整目录，而不是只吐出一段 Markdown：

```text
output/article-title/
├── article.md
├── metadata.json
├── extraction-report.json
└── images/
    ├── img_001.png
    └── img_002.png
```

`article.md` 给人看和继续编辑；`metadata.json` 给后续发布流程用；`extraction-report.json` 用来排查这篇文章为什么有图片、视频或格式问题。

## 适合现在就试的场景

- 把微信公众号文章备份成 Markdown，并保留图片和来源信息。
- 把掘金、CSDN 技术文章整理到本地知识库。
- 批量转换一组文章链接，然后人工检查 `batch-report.md`。
- 作为 Agent Skill，让 Agent 按固定流程完成文章采集和转换。

如果你的目标是登录后内容、付费内容、验证码页面、App-only 页面，v0.1.0 还不适合。

## 怎么开始

推荐用 `uv` 全局安装：

```bash
uv tool install magicmd
magicmd doctor
```

如果你习惯 `pipx`：

```bash
pipx install magicmd
magicmd doctor
```

参与开发或想使用当前源码时，再从仓库运行：

```bash
git clone https://github.com/didilili/MagicMD.git
cd MagicMD
uv sync --extra dev
uv run magicmd doctor
```

确认环境正常后，准备一个 `urls.txt`：

```text
https://mp.weixin.qq.com/s/example
https://juejin.cn/post/example
https://blog.csdn.net/user/article/details/123
```

批量转换：

```bash
magicmd batch urls.txt -o output/
```

重复跑同一批链接时，建议加上：

```bash
magicmd batch urls.txt -o output/ --skip-existing
```

如果你正在调格式，想强制重新生成：

```bash
magicmd batch urls.txt -o output/ --overwrite
```

## 这个版本重点解决了什么

### 微信公众号

微信公众号文章不是普通 HTML 页面，里面经常有懒加载图片、视频播放器、复杂富文本、推荐阅读区、二维码和排版用容器。v0.1.0 做了这些处理：

- 提取标题、作者、发布时间、正文和来源链接。
- 下载正文图片，并把 Markdown 图片链接改成本地路径。
- 提取视频链接，并尝试下载到本地；如果微信防盗链导致失败，会在报告里留下 warning。
- 清理一部分排版容器，避免把装饰图片或视觉布局误转成 Markdown 标题。
- 处理推荐阅读区、加粗碎片、块级链接边界和非代码 `pre` 文本。

### 掘金

掘金文章里常见的问题是外链会走 `link.juejin.cn` 风险跳转，复制到 Markdown 后不够好用。v0.1.0 会尽量把这类链接还原成真实目标地址，让 Markdown 里的链接可以直接打开。

这个版本也处理了掘金文章里的图片、代码块和标题层级。

### CSDN

CSDN 的重点问题是代码区。页面里经常混着复制按钮、行号、语言标签、展开控件和其它编辑器元素。v0.1.0 针对 10 篇复杂样本做过人工检查，重点修了：

- 代码块错位。
- 代码块里多出来的孤立数字。
- 高亮代码换行丢失。
- Mermaid/SVG 图转换异常。
- 站内目录链接无法跳转。
- 代码复制按钮和编辑器控件噪声。

## 和同类工具相比

MagicMD 不是要替代 MarkItDown、Trafilatura、Crawl4AI 这类更通用的工具。它更像一个偏中文内容平台的文章归档工具。

- 如果你要把各种文件转 Markdown，通用转换工具更合适。
- 如果你要做复杂爬虫、调度、抓取策略，爬虫框架更合适。
- 如果你主要处理微信公众号、掘金、CSDN 文章，并希望输出能进入自己的内容仓库，MagicMD 会更省事。

MagicMD 的输出不是一次性文本，而是内容包：正文、图片、元数据、报告放在一起。这个结构更适合后续接 GitHub、HaoGit、静态站点或 Agent 工作流。

## 批量报告怎么看

批量转换后会生成：

```text
output/
├── batch-report.json
└── batch-report.md
```

建议先看 `batch-report.md`。它会告诉你：

- 哪些链接成功了。
- 哪些链接失败了。
- 失败发生在 `detect`、`fetch`、`parse`、`write`、`media` 还是 `report`。
- 哪些文章有图片、视频或 Markdown 质量疑点。

如果某篇文章格式不对，先看它目录里的 `extraction-report.json`。如果启用了 debug HTML，再看 `debug.html`。

## 配置

先生成默认配置：

```bash
magicmd config init
```

常用配置包括：

- 输出目录。
- 是否覆盖已有内容包。
- 是否输出 YAML front matter。
- 是否保留来源信息块。
- 标题层级偏移。
- 图片目录和图片命名规则。
- 每个平台的抓取方式和等待选择器。
- Camoufox 浏览器等待超时和重试次数。

完整示例在 `.magicmd.example.toml`。

## 安全边界

MagicMD 只处理公开文章页面。它不会绕过登录、付费墙、私有内容、验证码或平台访问限制。

如果页面返回 403、需要登录、视频资源防盗链、动态资源过期，MagicMD 会尽量保留已经能提取的内容，并在报告里记录 warning 或失败原因。

## English Summary

MagicMD v0.1.0 is the first usable release of a CLI and Agent Skill for converting public article URLs into Markdown content packages.

It focuses on WeChat Official Account articles, Juejin, CSDN, and generic public article pages. The output includes `article.md`, `metadata.json`, `extraction-report.json`, and local media directories.

This release is especially tuned for Chinese content platforms: WeChat rich text and media handling, Juejin direct-link restoration, CSDN code-block cleanup, batch reports, configurable Markdown output, and Agent Skill integration.

MagicMD only targets public article pages. It does not bypass login, paywalls, private content, CAPTCHA, or platform access controls.
