# MagicMD v0.1.0

v0.1.0 是 MagicMD 的第一个可用版本。

这个版本先把一件事做稳：把公开文章链接转换成可以长期保存、继续编辑、接入发布流程的 Markdown 内容包。它不是通用爬虫框架，也不处理登录后内容；它的重点是微信公众号、掘金、CSDN 和普通公开文章页。

## 这版可以直接做什么

```bash
magicmd "https://mp.weixin.qq.com/s/example"
magicmd batch urls.txt -o output/
```

一次转换会生成一个完整目录，而不是只输出一段 Markdown：

```text
output/article-title/
├── article.md
├── metadata.json
├── extraction-report.json
└── images/
    ├── img_001.png
    └── img_002.png
```

`article.md` 用来阅读和继续编辑；`metadata.json` 给后续发布流程使用；`extraction-report.json` 用来排查图片、视频、正文结构或抓取阶段的问题。

## 支持的平台

- 微信公众号：提取标题、作者、发布时间、正文、来源链接和正文图片，并尽量处理懒加载图片、富文本容器、推荐阅读区和视频资源。
- 掘金：处理标题、正文、图片、代码块，并尽量还原 `link.juejin.cn` 跳转后的真实链接。
- CSDN：重点清理代码块里的行号、复制按钮、编辑器控件、语言标签和高亮换行问题。
- 通用网页：对普通公开文章页面做基础正文提取，适合简单博客和新闻页。

## 批量转换

准备 `urls.txt`：

```text
https://mp.weixin.qq.com/s/example
https://juejin.cn/post/example
https://blog.csdn.net/user/article/details/123
```

运行批量转换：

```bash
magicmd batch urls.txt -o output/
```

重复跑同一批链接时，可以跳过已存在的内容包：

```bash
magicmd batch urls.txt -o output/ --skip-existing
```

调试格式或重新生成时，可以显式覆盖：

```bash
magicmd batch urls.txt -o output/ --overwrite
```

批量转换后会生成 `batch-report.md` 和 `batch-report.json`，方便先看哪些链接成功、哪些失败，以及失败发生在 `detect`、`fetch`、`parse`、`write`、`media` 还是 `report` 阶段。

## 配置

生成默认配置：

```bash
magicmd config init
```

v0.1.0 已经支持配置输出目录、是否覆盖已有内容包、是否输出 YAML front matter、是否保留来源信息块、标题层级偏移、图片目录、图片命名规则，以及部分平台抓取参数。

完整示例见 `.magicmd.example.toml`。

## 安全边界

MagicMD 只处理公开文章页面。它不会绕过登录、付费墙、私有内容、验证码或平台访问限制。

如果页面返回 403、需要登录、视频资源防盗链、动态资源过期，MagicMD 会尽量保留已经能提取的内容，并在报告里记录 warning 或失败原因。

# MagicMD v0.1.0

v0.1.0 is the first usable MagicMD release.

It focuses on converting public article URLs into Markdown content packages. A conversion writes `article.md`, `metadata.json`, `extraction-report.json`, and local media files together, so the result can be archived, inspected, and passed into later publishing workflows.

## Highlights

- WeChat Official Account article extraction with image handling and source metadata.
- Juejin article conversion with direct-link restoration where possible.
- CSDN article conversion with extra cleanup for code blocks and editor noise.
- Batch conversion with Markdown and JSON reports.
- Configurable output behavior through `.magicmd.toml`.
- Clear boundary: public article pages only; no login, paywall, CAPTCHA, or private-content bypassing.
