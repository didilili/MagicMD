# MagicMD v0.1.1

v0.1.1 是一个发布体验修补版。核心转换能力延续 v0.1.0，这次重点让新用户更容易安装、排障，也让脚本和 Agent 更容易判断环境是否可用。

## 这次升级了什么

- 发布 npm wrapper，可通过 `npm install -g magicmd` 或 `npx magicmd` 获得全局命令。
- 优化 npm wrapper 在缺少 `uvx` 时的提示，直接给出 macOS/Linux、Windows、`uv tool install` 和 `pipx` 安装路径。
- 新增 `magicmd doctor --json`，方便 CI、脚本和 Agent Skill 读取诊断结果。
- 新增 `docs/troubleshooting.md`，集中排查安装、Camoufox、403、视频防盗链、图片失败和批量中断问题。
- 完善 `docs/supported-sites.md`，明确微信公众号、掘金、CSDN 和通用网页的支持边界。

## 安装

Python 用户推荐：

```bash
uv tool install magicmd
magicmd doctor
```

npm 用户可以使用：

```bash
npm install -g magicmd
magicmd doctor
```

如果你使用 npm 包，系统仍然需要能运行 `uvx`。npm 包只是一个轻量入口，底层调用 PyPI 版 MagicMD，避免 Python CLI、npm CLI 和 Agent Skill 各自维护一套解析逻辑。

## 验证环境

```bash
magicmd doctor
magicmd doctor --json
```

`doctor --json` 会输出机器可读的诊断结果，适合放进自动化脚本、CI 或 Agent workflow。

## 适合谁升级

- 想用 npm 全局安装 MagicMD 的用户。
- 安装或抓取遇到问题，需要更清楚排障提示的用户。
- 想在脚本、CI 或 Agent Skill 里判断 MagicMD 环境是否可用的用户。

# MagicMD v0.1.1

v0.1.1 is a release-experience patch. The conversion core stays aligned with v0.1.0; this release makes install, troubleshooting, and automation diagnostics easier.

## What's Changed

- Published the npm wrapper so users can run MagicMD via `npm install -g magicmd` or `npx magicmd`.
- Improved the npm wrapper message when `uvx` is missing, with macOS/Linux, Windows, `uv tool install`, and `pipx` options.
- Added `magicmd doctor --json` for CI, scripts, and Agent Skill diagnostics.
- Added `docs/troubleshooting.md` for install, Camoufox, 403, video hotlinking, failed images, and interrupted batch runs.
- Expanded `docs/supported-sites.md` with clearer boundaries for WeChat, Juejin, CSDN, and generic pages.

## Install

Recommended for Python users:

```bash
uv tool install magicmd
magicmd doctor
```

npm users can run:

```bash
npm install -g magicmd
magicmd doctor
```

The npm package still requires `uvx` on the system. It is a thin entrypoint that forwards to the PyPI MagicMD CLI, keeping the Python CLI, npm CLI, and Agent Skill on one shared conversion core.
