# MagicMD v0.1.1

v0.1.1 是一个发布体验修补版。

核心转换能力延续 v0.1.0，这一版重点处理安装入口、排障说明和自动化诊断。对刚开始使用 MagicMD 的用户来说，它让“装不上、环境不对、脚本不知道能不能跑”的问题更容易定位。

## 这版处理的事情

- 发布 npm wrapper，可以通过 `npm install -g magicmd` 或 `npx magicmd` 获得命令入口。
- 优化 npm wrapper 在缺少 `uvx` 时的提示，直接给出 macOS/Linux、Windows、`uv tool install` 和 `pipx` 安装路径。
- 新增 `magicmd doctor --json`，方便 CI、脚本和 Agent Skill 读取诊断结果。
- 新增 `docs/troubleshooting.md`，集中说明安装、Camoufox、403、视频防盗链、图片失败和批量中断问题。
- 完善 `docs/supported-sites.md`，明确微信公众号、掘金、CSDN 和通用网页的支持边界。

## 怎么使用

Python 用户推荐：

```bash
uv tool install magicmd==0.1.1
magicmd doctor
```

也可以用 `pipx`：

```bash
pipx install "magicmd==0.1.1"
magicmd doctor
```

npm 用户可以使用：

```bash
npm install -g magicmd@0.1.1
magicmd doctor
```

npm 包是轻量入口，底层仍然调用 PyPI 版 MagicMD。这样 Python CLI、npm CLI 和 Agent Skill 不需要维护多套转换逻辑。

## 自动化诊断

```bash
magicmd doctor
magicmd doctor --json
```

`doctor --json` 会输出机器可读的诊断结果，适合放进 CI、脚本或 Agent workflow。

# MagicMD v0.1.1

v0.1.1 is a release-experience patch.

The conversion core stays aligned with v0.1.0. This release improves installation paths, troubleshooting docs, and machine-readable diagnostics for scripts, CI, and agent workflows.

## What Changed

- Published the npm wrapper for `npm install -g magicmd` and `npx magicmd`.
- Improved the missing-`uvx` message with clear install options.
- Added `magicmd doctor --json`.
- Added troubleshooting documentation for install and runtime issues.
- Clarified supported-site boundaries for WeChat, Juejin, CSDN, and generic pages.

## Install This Version

```bash
uv tool install magicmd==0.1.1
# or
pipx install "magicmd==0.1.1"
# npm wrapper
npm install -g magicmd@0.1.1
```
